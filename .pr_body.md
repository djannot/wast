## Summary

Implements the API security testing features advertised in `wast api` help text. This PR adds comprehensive security testing capabilities including JWT token analysis, BOLA/IDOR detection, SQL injection testing, mass assignment vulnerability detection, and authentication bypass testing.

Closes #230

## Features Implemented

### 1. **JWT Token Analysis** (`pkg/api/jwt.go`)
- Extracts JWT tokens from Authorization headers (case-insensitive)
- Decodes and analyzes tokens without signature verification (intentional for security testing)
- Detects critical vulnerabilities:
  - `alg: none` attacks
  - Missing signatures
  - Expired tokens
  - Sensitive data in payloads (email, password, ssn, etc.)
- Warns about weak algorithms (HS256)
- Checks standard claims (exp, iat, nbf)

### 2. **BOLA/IDOR Vulnerability Testing** (`pkg/api/security_tester.go`)
- Tests path parameters by manipulating resource IDs
- Tries multiple test values: `1`, `2`, `999`, `0`, `admin`
- Detects unauthorized access to resources
- Includes confidence levels based on endpoint security requirements
- Reports severity, description, and remediation guidance

### 3. **SQL Injection Testing**
- Integrates existing SQLi scanner from `pkg/scanner/sqli.go`
- Tests query parameters with SQL injection payloads
- Reports findings with proper severity classification
- Provides detailed remediation guidance

### 4. **Mass Assignment Vulnerability Detection**
- Tests if extra fields can be modified (role, admin, is_admin, is_staff, permissions, etc.)
- Checks if sensitive fields are accepted in POST/PUT/PATCH requests
- Includes confidence level assessment (marked as "low" per review feedback)
- Helps identify improper field whitelisting

### 5. **Authentication Bypass Testing**
- Tests if protected endpoints are accessible without authentication
- Compares responses with and without auth headers
- Detects endpoints that should require auth but don't

### 6. **Integration with Existing API Tester**
- Seamlessly integrated via functional options pattern
- Enabled by default, can be disabled with `WithSecurityTesting(false)`
- Respects rate limiting to avoid overwhelming servers
- Context-aware for proper cancellation support
- Properly closes all HTTP response bodies

## Notes

- Security testing is **enabled by default** for the API tester
- All security tests respect rate limiting configuration
- JWT analysis is for security testing purposes only (does not verify signatures)
- BOLA detection includes confidence levels to reduce false positives
- Mass assignment detection is marked as "low" confidence and requires manual verification

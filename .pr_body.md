## Summary

This PR verifies and documents that issue #118 (Add machine-readable JSON output format to CLI commands) is **already fully implemented** in the codebase. No code changes were required.

Closes #118

## Verification Results

### ✅ Feature Already Implemented

The `--output` / `-o` flag is **already functional** on all scanning commands with the following options:
- `text` (default) - Human-readable output
- `json` - Machine-readable JSON format
- `yaml` - Machine-readable YAML format
- `sarif` - SARIF 2.1.0 format for security tools

### ✅ Implementation Details

1. **Global Flag Definition** (`cmd/wast/root.go:165-166`)
   - Persistent flag inherited by all subcommands
   - Validates format before execution

2. **Output Formatter** (`pkg/output/formatter.go`)
   - Implements `FormatJSON`, `FormatYAML`, `FormatText`, `FormatSARIF`
   - Provides consistent `CommandResult` structure with `success`, `command`, `message`, `data` fields

3. **All Commands Use Formatter**
   - `scan` - ✅ Uses formatter via `getFormatter()`
   - `recon` - ✅ Uses formatter via `getFormatter()`
   - `crawl` - ✅ Uses formatter via `getFormatter()`
   - `api` - ✅ Uses formatter via `getFormatter()`
   - `intercept` - ✅ Uses formatter via `getFormatter()`

4. **Result Structs Have JSON/YAML Tags**
   - `HeaderScanResult`, `SQLiScanResult`, etc. in `pkg/scanner/`
   - All structs properly tagged for serialization

5. **Documentation is Complete**
   - CLI reference (`docs/cli-reference.md`) documents `--output` flag
   - Help text includes examples with `--output json` and `-o yaml`
   - Output structure examples provided for each command

6. **Tests Exist and Pass**
   - `pkg/output/formatter_test.go` - Full test coverage
   - All 97.7% test coverage maintained
   - Tests verify JSON, YAML, text, and SARIF output

### Example Usage (Already Working)

```bash

wast scan https://example.com -o json

# YAML output

wast scan https://example.com --output sarif > results.sarif
```
